# Documentação

Propósito: concentrar o conhecimento do toolkit (padrões, guias de uso, referências rápidas) para facilitar onboarding e consistência.

Organização sugerida:
- `docs/guides/`: guias passo a passo (ex.: como criar uma medida, como validar um dataset, como publicar um dashboard).
- `docs/references/`: referência rápida (naming conventions, checklist de qualidade, padrões de SQL/DAX).
